## 議題: guide-siteのSEO最強化

---

### 🦊 佐藤マコト
皆さん、今日はguide-siteのSEO最強化について話し合いましょう。現状を分析したところ、いくつかの改善ポイントが見つかりました。

**現状の問題点:**
1. メタタグが全ページ共通になっている
2. OGP（Open Graph Protocol）が未設定
3. 構造化データ（JSON-LD）がない
4. サイトマップがない
5. robots.txtがない
6. パンくずリストがない

---

### 🐱 田中ユイ
なるほど。Next.js 15なら、これらの多くはMetadata APIで簡単に実装できますね。

**技術的なアプローチ:**
```typescript
// 各ページでメタデータを個別設定
export const metadata: Metadata = {
  title: "第1章 Xアルゴリズムの全体像",
  description: "推薦システムの設計思想から...",
  openGraph: {
    title: "...",
    description: "...",
    images: ["/og-image.png"],
  },
};
```

サイトマップは `app/sitemap.ts` で動的生成できます。

### 🦊 佐藤マコト
構造化データは特に重要です。Googleのリッチリザルトに表示される可能性が上がります。

**推奨する構造化データ:**
1. **WebSite** - サイト全体の情報
2. **Article** - 各章をArticleとしてマークアップ
3. **BreadcrumbList** - パンくずリスト
4. **FAQPage** - 各章のFAQセクション

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "第1章 Xアルゴリズムの全体像と思想",
  "author": {
    "@type": "Organization",
    "name": "X Algorithm Guide"
  }
}
```

### 🐻 鈴木ケンタ
内部リンクの設計も見直したいな。

**内部リンク改善案:**
1. 各章の最後に「関連する章」を表示
2. 用語集から該当する章へリンク
3. 章の途中で関連概念へのリンク

例えば「Phoenix ML」という単語が出てきたら、第4章へリンクするとか。
