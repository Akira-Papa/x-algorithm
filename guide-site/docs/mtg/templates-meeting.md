### 🐱 田中ユイ
技術的な実装を考えると、こういう構成がいいと思う。

**ページ構成:**
```
/templates                    # テンプレート一覧ページ
/templates/[category]         # カテゴリ別一覧（14ページ）
/templates/[category]/[id]    # テンプレート詳細（140ページ）
```

**テンプレート詳細ページの構成:**
1. テンプレート名
2. カテゴリバッジ
3. 狙える効果（アルゴリズム的メリット）
4. 例文（コピー可能）
5. 解説
6. ポイント（3-5個）
7. 注意点（2-3個）
8. 参考ガイドリンク
9. 関連テンプレートへのリンク

### 🦉 山田リン
データ構造を整理しよう。

**テンプレートデータ構造:**
```typescript
interface Template {
  id: string;                    // "engagement-01"
  category: CategoryId;          // "engagement"
  title: string;                 // "共感を呼ぶ自己体験シェア"
  description: string;           // 短い説明
  effects: Effect[];             // 狙える効果
  example: string;               // 例文
  explanation: string;           // 解説
  points: string[];              // ポイント
  cautions: string[];            // 注意点
  relatedChapters: number[];     // 関連する章番号
  relatedTemplates: string[];    // 関連テンプレートID
  difficulty: 'beginner' | 'intermediate' | 'advanced';
  expectedEngagement: {
    likes: 'low' | 'medium' | 'high';
    replies: 'low' | 'medium' | 'high';
    reposts: 'low' | 'medium' | 'high';
    bookmarks: 'low' | 'medium' | 'high';
  };
}

interface Category {
  id: string;
  name: string;
  description: string;
  icon: string;
  targetEngagement: string[];
  templateCount: number;
}
```

**SEO対策:**
- 各テンプレートに固有のtitle/description
- 構造化データ（HowTo, Article）
- カテゴリページにも適切なメタデータ
- パンくずリスト完備
